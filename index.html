Olá, eu sou o Felipe

Sempre fui fascinado pela forma como a tecnologia conecta as pontas. Iniciei minha jornada desenvolvendo aplicações no Frontend e Backend, compreendendo todo o ciclo de vida do desenvolvimento de software.

Hoje, percebo que criar um bom código é apenas metade do caminho. Minha verdadeira paixão brilha ao desenhar o ambiente onde esse código roda. Por isso, estou direcionando meu foco para Cloud Infrastructure, com especialização em AWS.

Estudo diariamente práticas de Infraestrutura como Código (IaC), deploy contínuo (CI/CD) e arquiteturas escaláveis e resilientes na nuvem da AWS.

Edge Service

Amazon CloudFront

Distribuição de conteúdo global (CDN) com baixíssima latência e alta velocidade de transferência de dados.

Minha proficiência neste recurso:

  • Configuração de distribuições HTTPS seguras vinculadas com AWS Certificate Manager (ACM).
  • Otimização de cache utilizando políticas customizadas e invalidação automática de cache via CI/CD.
  • Origens configuradas dinamicamente apontando para S3 (site estático/assets) e Application Load Balancer (para a API backend).
Código Terraform Recomendado:
resource "aws_cloudfront_distribution" "s3_distribution" {
  origin {
    domain_name = aws_s3_bucket.portfolio_bucket.bucket_regional_domain_name
    origin_id   = "S3-Portfolio"
  }
  enabled             = true
  default_root_object = "index.html"
}
Networking

Application Load Balancer (ALB)

Roteador inteligente de requisições HTTP/HTTPS entre múltiplas instâncias na VPC, distribuindo o tráfego de forma uniforme.

Minha proficiência neste recurso:

  • Criação de Target Groups com Health Checks inteligentes para evitar envio de requisições a servidores offline.
  • Associação com políticas de HTTPS para SSL Offloading e redirecionamentos automáticos de HTTP para HTTPS (porta 80 -> 443).
  • Distribuição de tráfego de entrada em múltiplas Zonas de Disponibilidade (Availability Zones).
Compute

Amazon EC2 & Auto Scaling

Capacidade computacional escalável na nuvem através de servidores virtuais redimensionáveis de acordo com a demanda real.

Minha proficiência neste recurso:

  • Provisionamento de instâncias utilizando Launch Templates declarativos e scripts de inicialização (User Data) automatizados.
  • Configuração de políticas de Auto Scaling dinâmicas baseadas no consumo de CPU e memória.
  • Segurança rigorosa com Security Groups estruturados sob o princípio do privilégio mínimo de conexões de rede.
Compute (Serverless)

AWS Lambda

Execução de código orientada a eventos sem a necessidade de provisionar ou gerenciar servidores físicos ou virtuais.

Minha proficiência neste recurso:

  • Desenvolvimento de funções serverless em Node.js com empacotamento enxuto de dependências.
  • Integração nativa com API Gateway para expor endpoints REST de forma performática e econômica.
  • Garantia de segurança utilizando Roles dedicadas do IAM com acessos restritos a recursos específicos (como S3 e DynamoDB).
Database

Amazon RDS Multi-AZ

Serviço de Banco de Dados Relacional gerenciado de fácil configuração, operação e escalabilidade na nuvem.

Minha proficiência neste recurso:

  • Provisionamento de instâncias PostgreSQL e MySQL com implantação Multi-AZ para alta disponibilidade e failover automático.
  • Políticas de backup agendadas com retenção de snapshots e criptografia de dados em repouso com chaves AWS KMS.
  • Isolamento de banco em subredes privadas dedicadas sem acesso direto pela internet.
Storage

Amazon Simple Storage Service (S3)

Armazenamento de objetos seguro, altamente escalável e de altíssima durabilidade (99.999999999% de durabilidade).

Minha proficiência neste recurso:

  • Hospedagem estática de sites seguros combinados com políticas rígidas de Bucket Policies e bloqueio de acesso público padrão.
  • Políticas de Ciclo de Vida (Lifecycle Rules) inteligentes para transição de classes de armazenamento baratas (Glacier) para otimização de custos.
  • Configuração de versionamento de objetos para prevenção de deleções acidentais.
